  • Existen diferentes ciclos de vida o hooks a lo largo de la vida de un componente
  • El hook ngOnChanges se dispara cuando los datos vinculados a un componente cambian de alguna forma
  • El hook ngOnInit se dispara cuando el componente está inicializándose o después del primer ngOnChanges
  • El hook ngDoCheck se dispara cuando se realiza una revisión del ciclo de detección de cambios, es decir, lo mismo que el anterior, pero aplicado a todo el contenido de la pantalla, no solo referente a un componente
  • El hook ngAfterContentInit se dispara después de insertar contenido
  • El hook ngAfterContentChecked se dispara después revisar el contenido insertado (supongo que después del ngAfterContentInit)
  • El hook ngAfterViewInit se dispara después de inicializar el componente o los componentes hijos
  • El hook ngAfterViewChecked se dispara después de revisar cada uno de los componentes hijos
  • El hook ngOnDestroy se dispara cuando el componente se destruye, bien cuando cambiamos de página o bien cuando se ejecuta la directiva ngIf si esta afecta al componente en cuestión (supongo que por ese mismo motivo ngSwitch debiera generar este hook, ya que es un ngIf más potente), entre otros
  • En el TS de este componente se encuentra implementado cada hook con un console log
  • Si abrimos la consola de JS podremos ver el orden de ejecución
  • Aunque cambiemos el orden dentro del código en el que aparecen las funciones que implementan los hooks, el orden de impresión por consola no variará
  • ¿Por qué? Porque cada hook tiene su momento de ejecución
  • Curiosamente el hook ngOnDestroy no se imprime por consola, dado que el componente no se ha destruido todavía
  • Para que este aparezca, debemos destruir el componente, por ejemplo cambiando de página
  • Si creas un botón que modifique el renderizado de un componente, se ejecutarán, en este orden, los hooks ngDoCheck, ngAfterContentChecked y ngAfterViewChecked
