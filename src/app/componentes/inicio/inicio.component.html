  • Me he formado en el funcionamiento de este framework durante un largo periodo de tiempo y he creado estas notas por si le pudieran servir a alguien basándome en diferente fuentes

Introducción

  • Angular es un framework de front desarrollado por Google que utiliza TypeScript
  • Un framework es un set de herramientas software que el programador tiene a su disposición para elaborar aplicaciones web contando con un gran abanico de prestaciones que no tendrías programando desde cero
  • TypeScript (de ahora en adelante, TS) es un lenguaje de programación desarrollado por Microsoft que utiliza JavaScript (de ahora en adelante, JS) como base crear un lenguaje de programación más robusto
  • Ofrece numerosas prestaciones adicionales con las que JS no cuenta, por lo que se conoce como un superconjunto de JS, como tipado de variables, notificación de errores previa al periodo de ejecución, etc.

  • Desde el directorio de nuestro proyecto ejecutamos el comando & npm install jquery --save para instalar jQuery dentro del directorio node_modules
  • Abrimos el fichero angular.json y dentro del campo vectorial scripts escribimos la ruta del fichero de distribución de jQuery dentro de este directorio
  • Es conveniente elegir el fichero .min al ser más ligero. La ruta típicamente suele ser "./node_modules/jquery/dist/jquery.min.js"
  • Tras esto, ejecutamos el comando $ npm install o solamente $ npm i para actualizar las dependencias
  • Dentro del componente en el que deseemos utilizar jQuery, escribimos la instrucción declare var $: any; delante del decorador
  • Utilizamos jQuery dentro del hook OnInit, de modo que lo incluimos en el componente en caso de haberlo quitado (Angular CLI lo crea por defecto)
  • Dentro podemos definir todos los manejadores en jQuery con total normalidad como hemos hecho siempre

Selector $(this)

  • Utilizar $(this) puede dar problemas, ya que en TypeScript no apunta al elemento HTML que disparó el evento, como solía suceder típicamente cuando programábamos sin utilizar Angular
  • En su lugar pasamos el evento como parámetro del manejador (event) y utilizamos event.currentTarget, que nos da acceso a todos los atributos necesarios del elemento HTML que disparó el evento
  • Debemos tener en cuenta que el evento puede tener cualquier identificador, pero comúnmente se le suele asignar event o e, pero no es obligatorio
  • Ante cualquier duda sobre la estructura del objeto event, siempre podemos inspeccionarla en la consola de JavaScript mediante la instrucción console.log(event)

Funciones flecha

  • Es posible simplifcar la instrucción del manejador de un evento asociado a un elemento HTML gracias a las funciones flecha
  • Por ejemplo, en lugar de $( '.ejemplo' ).click( function( ... ) { ... } ), podemos escribir $( '.ejemplo' ).click( ( ... ) => { ... } ) y omitir así la palabra reservada function
